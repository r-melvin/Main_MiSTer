# Launcher Simplification: Drop Network Scraping, Replace CSVs with Directory Walk, Split and Test

## Context

The `feature/ui-launcher-overhaul` branch added a launcher that originally pulled ROMs from a remote server and scraped cover art + game descriptions from TheGamesDB. The remote-pull feature has been removed, but two leftovers remain:

1. **CSV-driven library loading.** `launcher_load_library` reads `{base}/lists/<System>.csv` files which used to be generated by the remote scanner. Now nothing generates them, so a fresh install shows the "no games" splash forever.
2. **Network scraping via curl.** `scrape_tgdb_cover` (~150 LOC) and `scrape_tgdb_description` (~200 LOC) call out to TheGamesDB for art + text. Also surfaces as the batch-download modal (`LMODE_COVER_DL`) and the `I`-key description overlay (`LMODE_DESCRIPTION`). Out of scope for an offline launcher.

Alongside those, [launcher_io.cpp](launcher_io.cpp) has grown to ~1,600 lines with five separate concerns (state, theme, cover cache, launch/MGL, remnant scanner). The test suite covers core-map lookup, cover cache LRU, state I/O basics, pause-core SPI, and the new `wrap_index` — none of the file scanning, none of the launch pipeline.

The goal is an offline, directory-driven launcher sharing ROMs with MiSTer's stock menu (`/media/fat/games/<System>/`), with its code split by concern and unit tests covering everything that doesn't need SDL/Imlib.

## Scope

**In scope:** remove curl-based scraping; remove `LMODE_COVER_DL` + `LMODE_DESCRIPTION` modes; implement a directory scanner that replaces CSV parsing; add a stat-invalidated on-disk cache; split `launcher_io.cpp` into focused modules; add tests for the scanner, state I/O, and launch pipeline.

**Out of scope:** splitting `launcher.cpp` (state machine stays monolithic); new themes; cover image format support beyond JPG/PNG as already handled; automated tests of the SDL/Imlib rendering or input state machine; any network feature (covers become BYO).

## Component design

### `launcher_scan` — directory scanner (new module)

Pure file walker. No globals, no threading inside the scanner itself — the existing `lib_loader_thread` calls it per system.

```c
// launcher_scan.h
const char * const *launcher_rom_extensions(const char *system);
int  launcher_scan_system(const char *system,
                          const char *games_dir,   /* "/media/fat/games" */
                          const char *covers_dir,  /* "<base>/covers" */
                          LauncherGame **out);
bool launcher_scan_cache_load(const char *cache_path,
                              const char *games_dir,
                              LauncherSystem **systems_out, int *count_out);
bool launcher_scan_cache_save(const char *cache_path,
                              const char *games_dir,
                              const LauncherSystem *systems, int count);
```

**Extension allowlist.** Per-system, sentinel-terminated:

| System | Extensions |
|---|---|
| SNES | sfc, smc |
| NES | nes |
| GB/GBC | gb, gbc |
| GBA | gba |
| Genesis / MegaDrive | md, gen, bin, smd |
| Sega32X | 32x |
| MasterSystem | sms |
| GameGear | gg |
| PCEngine / TurboGrafx16 | pce, sgx |
| NeoGeo | neo |
| Arcade | mra |
| Atari2600 | a26, bin |
| Atari7800 | a78, bin |
| AtariLynx | lnx |
| ColecoVision | col, bin |
| Intellivision | int, bin |
| PSX | chd, cue, exe |
| N64 | n64, z64, v64 |
| C64 | prg, d64, t64, tap, crt |
| AmigaOCS | adf, hdf |

These derive from the file extensions MiSTer's core-specific loaders already accept; stray files get filtered out.

**Walk semantics.** Recursive, depth-first, via `opendir`/`readdir` + a small stack (no `nftw` — avoids pulling in ftw.h symbols and keeps tests portable). Skips entries starting with `.`. Skips symlinks pointing outside `games_dir`. Follows the game's on-disk name verbatim for `LauncherGame.name` (strip extension only) — no sanitisation on read.

**Cover resolution.** For each game, `covers_dir/<System>/<stem>.jpg` and `.png` probed in order. First match stored in `LauncherGame.cover_path`; miss stores empty string and the UI shows the procedural placeholder.

### `launcher_scan` — cache format

One binary file: `<base>/cache/library.bin`.

```
[Header]
  magic            uint32   'MSTR'
  version          uint32   2
  games_dir        char[512]
  system_count     uint32
  [Per-system]
    name           char[64]
    mtime          int64    (st.st_mtim.tv_sec of games_dir/<name>)
    game_count     uint32
    [Per-game]
      name, path, cover_path, system : fixed-size strings as in LauncherGame
      last_played, play_count, play_time, file_size, user_rating
```

**Load path**: `launcher_scan_cache_load` reads the header, rejects on magic/version mismatch or path mismatch, then stats each system's directory. If any `st_mtime > cached_mtime` the loader returns `false` to signal "rescan needed". The caller rescans missing/stale systems and calls `launcher_scan_cache_save` afterwards.

**Save path**: atomic — write `library.bin.tmp` then `rename()`.

**Corruption handling**: any read error → log + return false → full rescan.

### Existing modules affected

`launcher_io.cpp` splits into five files, each with its own header-level section exposed via existing [launcher_io.h](launcher_io.h) or new headers:

- `launcher_io.cpp` (~400 LOC) — state I/O (favourites, recent, play_time), core-map table, `launcher_path` helpers. Keeps the existing public API.
- `launcher_scan.cpp` (new, ~350 LOC) — scanner + cache described above.
- `launcher_theme.cpp` (new, ~130 LOC) — `theme_set_defaults`, `launcher_theme_init`, `launcher_theme_load`, `parse_color`, `parse_font_size`.
- `launcher_cover.cpp` (new, ~500 LOC) — cover worker thread, LRU cache, `launcher_cover_*` API. **Drops** the lazy-scrape branch and its `strstr`/`exec_curl` call.
- `launcher_launch.cpp` (new, ~200 LOC) — `launcher_write_mgl`, `find_rbf`, `launcher_load_core`, `launcher_classify_launch_error`, `launcher_start_download` (local-symlink variant), `launcher_poll_download`, `launcher_cancel_download`.

Headers mirror the split: `launcher_scan.h`, `launcher_theme.h`, `launcher_cover.h`, `launcher_launch.h`. Existing `launcher_io.h` shrinks to state + core-map.

[launcher.cpp](launcher.cpp) changes: drop `LMODE_COVER_DL`/`LMODE_DESCRIPTION` from the enum, `enter_cover_dl`, `toggle_multi_mode` review, `g_cdl_*` state, `g_desc_*` state, `KEY_D` and `KEY_I` handlers, their `draw_*` cases, and the description-modal footer hint. `lib_loader_thread` calls the new cache-aware load path instead of `launcher_load_library`.

[cfg.h](cfg.h) / [cfg.cpp](cfg.cpp): drop `tgdb_api_key`; add `launcher_games_path[1024]` defaulting to `"/media/fat/games"`. INI key `LAUNCHER_GAMES_PATH`.

[Makefile](Makefile): no link change (curl is invoked via `execvp`, not linked). Just add the new `.cpp` files to the source list.

[README.md](README.md) / [launcher.cfg.example](launcher.cfg.example): drop TGDB section; add `LAUNCHER_GAMES_PATH` doc; note that covers are now BYO (`<base>/covers/<System>/<stem>.jpg`).

## Data flow

Boot / refresh:

```
launcher_init()
  → lib_loader_thread()
      → launcher_scan_cache_load(library.bin) ?
          hit  → done (publish progress as "cached")
          miss → for each system in core_map:
                   stat games_dir/<system>/
                   launcher_scan_system(...)
                   g_load_done++
                 launcher_scan_cache_save(library.bin)
      → build virtual systems (Favourites, Recent, All Games)
      → g_lib_done = true
```

Launch: unchanged except `launcher_classify_launch_error` + `launcher_write_mgl` now live in `launcher_launch.cpp`.

## Error handling

- `games_dir` missing → cache load fails, rescan opens dir, fails cleanly, library empty → existing "no games" splash is shown.
- `games_dir/<System>/` missing → single system gets 0 games; other systems unaffected.
- Unreadable entry inside a system dir → skip + log; continue.
- Cache file partially written (power loss mid-write) → magic/header check fails → full rescan on next boot.
- Clock skew producing future mtime → accept at face value; only behavioural effect is an unnecessary rescan next boot if mtime drifts backwards.

## Testing

Three new / expanded test binaries, each self-contained (no SDL/Imlib):

**[tests/test_launcher_scan.cpp](tests/test_launcher_scan.cpp)** (new, ~300 LOC)
- Extension allowlist: SNES dir with `.sfc`, `.smc`, `.txt`, `.jpg` → 2 games.
- Recursive walk: `SNES/USA/Action/mario.sfc` is found.
- Empty dir → 0 games, no error.
- Missing dir → returns -1 (error), doesn't crash.
- Cover resolution: `<stem>.jpg` and `<stem>.png` both resolve; `<stem>.bmp` does not.
- Cache round-trip: save then load returns identical contents.
- Cache mtime invalidation: save, touch a system dir, load → returns false (rescan needed).
- Cache version mismatch: write a v1 header → load → returns false.
- Cache corrupted: truncate file mid-header → load → returns false, no crash.
- `games_dir` path change invalidates cache.

**[tests/test_launcher_state.cpp](tests/test_launcher_state.cpp)** (new, ~200 LOC — split from existing test_launcher_io)
- State round-trip with 1k recent entries.
- Corrupted state file → load returns empty state, no crash.
- Merge sequence: add fav, save, load, remove, save, load → deterministic.
- `play_time` accumulation across simulated sessions.
- Sort orders: A-Z, Z-A, Recent, Most Played, Top Rated on a known fixture.

**[tests/test_launcher_launch.cpp](tests/test_launcher_launch.cpp)** (new, ~180 LOC)
- `launcher_write_mgl` produces expected XML for a console game: `<rbf>`, `<file delay>`, `<reset>` in correct order.
- Arcade path writes the MRA as the `<file>` target (no `<rbf>` element — MRA loads the core itself).
- Missing core: fake `/media/fat/` with no `.rbf` → classifier returns "Core 'SNES' not found on SD card (looked in /media/fat/_Console/)."
- Virtual system (Favourites) fallthrough → classifier returns generic message.
- No core mapping → specific "No core is configured for the 'Foo' system." message.

**Existing tests retained as-is**: `test_launcher_core_map`, `test_launcher_covers`, `test_launcher_io` (shrinks to non-state bits like `launcher_cache_path`), `test_launcher_nav`, `test_pause_core`.

Target: ~80 tests across the suite (up from 69). All run via `make -C tests` and exit 0.

## Commit plan

Each commit is individually build-green and test-green.

1. `launcher: remove TheGamesDB cover + description scraping` — deletes `exec_curl`, `scrape_tgdb_cover`, `scrape_tgdb_description`, `tgdb_platform_id`, lazy-scrape branch in cover worker, `g_desc_*` state, the whole description-fetch thread and `LMODE_DESCRIPTION`. Keeps disk-loaded covers working.
2. `launcher: remove batch cover download modal` — drops `LMODE_COVER_DL`, `enter_cover_dl`, `g_cdl_*`, `launcher_draw_cover_dl`, `KEY_D` handler.
3. `launcher: add directory scanner and per-system extensions` — new `launcher_scan.{cpp,h}` + tests. Not wired into production yet; `launcher_load_library` still reads CSVs.
4. `launcher: add stat-invalidated library cache` — extends `launcher_scan` with `cache_load`/`cache_save`. Tests.
5. `launcher: switch library load to directory scanner` — `lib_loader_thread` now calls scanner+cache; delete `parse_csv` and old `launcher_load_library`. Add `LAUNCHER_GAMES_PATH` cfg + default. README updates.
6. `launcher: split launcher_io.cpp into theme/cover/launch modules` — mechanical move; no logic change. Headers tidied.
7. `launcher: expand test suite — state, launch, sort` — new/expanded tests matching the split.

Risk-ordered: #1 and #2 are pure deletion (large but mechanical). #3–#4 add code without changing behaviour. #5 is the behavioural flip. #6 is mechanical refactor. #7 strengthens the safety net.
